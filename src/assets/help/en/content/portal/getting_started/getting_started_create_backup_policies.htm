Create Backup Policies

Backup policies, which are also referred to as service level agreement (SLA) policies, define parameters that are applied to backup jobs. These parameters include the frequency and retention of backups.

About this task

The three default SLA policies are Gold, Silver, and Bronze. You can use these policies as they are or modify the policies. You can also create custom SLA policies.

If a virtual machine is associated with multiple SLA policies, ensure that the policies are not scheduled to run concurrently. Either schedule the SLA policies to run with a significant amount of time between them, or combine them into a single SLA policy.

For example purposes, this task does not include instructions for enabling replication for vSnap servers or for offloading to IBM Spectrum Protectâ„¢, which are optional features. For information about how to set up these features in the SLA policy, see Configure SLA Policies.

Backup copies of data are called snapshots.

Procedure

To create an SLA policy, complete the following steps:

  1. In the navigation tree, click Management Protection, and then click Policy Overview.
  2. Click Add SLA Policy. The New SLA Policy pane opens.
  3. In the Name field, enter a name that provides a meaningful description of the SLA policy.
  4. In the Operational Protection section under Main Policy, set the following options for backup operations. These operations occur on the vSnap servers that are defined in the System Configuration > Backup Storage > Disk window.
  5. Retention
  6. Specify the retention period or the number of snapshots to keep.
  7. Frequency
  8. Enter the frequency for backup operations.
  9. Start Time
  10. Enter the date and time that you want the backup operation to start.
  11. Target Site
  12. Select a primary or secondary backup destination.
  13. If more than one primary or secondary storage site is available to IBM Spectrum Protect Plus, the vSnap backup destination with the largest amount of available storage is used first.
  14. Only use encrypted disk storage
  15. Select this check box to back up data to encrypted vSnap servers if your environment includes a mixture of encrypted and unencrypted servers.
  16. RESTRICTION: If this option is selected and no encrypted vSnap servers are available, the associated job will fail.
  17. Click Save. The SLA policy can now be applied to backup job definitions as shown in Create a Backup Job.
