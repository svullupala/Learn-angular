Detecting Db2 Data by Running an Inventory Job

After you add your IBM® Db2® instances to IBM Spectrum Protect™ Plus, an inventory to detect all Db2 instances and databases is run automatically. You can run an inventory on any instance manually to detect, list and store all Db2 databases for the selected instance making them available for protection with IBM Spectrum Protect Plus.

BEFORE YOU BEGIN:

  • Ensure that you added your Db2 instances to IBM Spectrum Protect Plus. For instructions about adding instances, see Add a Db2 Server.

To detect Db2 data by running an inventory job: 

  1. From the navigation menu, expand Manage Protection, then expand Applications. Expand Db2, then click Backup.
  2. Select the check box beside the instance names that you want to run inventories for.
  3. Click Run Inventory.
  4. When the inventory is running, the button changes to Inventory In Progress. You can run an inventory on any available instance, but you can run only one inventory process at a time. To monitor the inventory job, select Jobs and Operations in the navigation, Monitoring > Job Monitor. The job is listed as Default Application Server Inventory.
  5. Click on an instance to open a view that shows the databases that are detected for that instance. If any databases are missing from the Instances list, check your Db2 instance and rerun the inventory.
  6. Tip: To return to the list of instances, click the Instances hypertext in the breadcrumbs of the pane.

NEXT STEPS:

  • To start protecting Db2 databases that are cataloged in the selected instance, apply an SLA policy to the instance. For instructions about setting an SLA policy, see Configure SLA Policies.
