Backing Up Db2 Databases

Define regular Db2Â® backup jobs with options to run and create backup copies to protect your data. You can enable continuous backing up of archive logs so that you can restore a point-in-time copy with rollforward options if required.

There are predefined SLA options to choose from as follows, Gold, Silver, and Bronze. Each specifies preset frequencies and retention rates.

Gold

Gold is the most frequent running every 4 hours with a retention of 1 week.

Silver

Silver runs daily with a retention of 1 month.

Bronze

Bronze runs daily with a retention of 1 week.

To back up Db2 databases:

  1. From the navigation menu, expand Manage Protection, then expand Applications. Expand Db2, then click Backup.
  2. Click the database or instance you want to back up by choosing one of the following choices.
    • Select an entire instance to back up in the Instances pane by clicking the check-box beside the instance name.
    • Select a specific database in an instance by clicking the instance name, and choosing a database from the list of databases in that instance.
  3. Each item in the Instances pane, is listed by instance or database name, the applied SLA policy, and the eligibility for log backup.
  4. Click Select Options to enable or disable log backup, and to specify parallel streams to maximize data movement in the backup operation. Save the options.
  5. Select Enable Log Backup to back up logs, which allows point-in-time restore options and recovery options. For Db2 log backup settings information, see Log backups.
  6. When you save the options, those options are used for all backup jobs for this database or instance as selected.
  7. Select the database or instance again, and click Select SLA Policy to choose an SLA policy for the selected database to instance.
  8. Save the SLA options.
  9. To define a new SLA or to edit an existing policy with custom retention and frequency rates, select Manage Protection > Policy Overview. In the SLA Policies pane, click Add SLA Policy, and define you policy preferences.

NEXT STEPS:

Once the SLA policy is created, you can choose to run the policy outside of the schedule by clicking Actions beside the policy name and selecting Start. The status in the log changes to show that the backup is Running.

  • Once your Db2 databases are listed for each of your Db2 instances, select and apply an SLA to start protecting your data. See Defining an SLA Backup Job.
  • When you are backing up data, you must select the option Enable Log Backups to enable log file backup. Archive log backup is only possible when either LOGARCHMETH1 or LOGARCHMETH2 is set to OFF. In some cases, a combination of settings can cause an error where IBM Spectrum Protect Plus cannot enable log archive backup. See Log backups.
