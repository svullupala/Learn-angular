Log backups

When you are backing up data, you must select the option Enable Log Backups to enable log file backup. Archive log backup is only possible when either LOGARCHMETH1 or LOGARCHMETH2 is set to OFF. In some cases, a combination of settings can cause an error where IBM Spectrum Protect™ Plus cannot enable log archive backup.

When you are backing up data with the Enable Log Backups selected, the IBM Spectrum Protect Plus Db2® agent looks for an available LOGARCHMETH parameter with an OFF value. This parameter is then used to set the DISK path for the backup operation.

Important: To ensure that your backup jobs include log archives, one of the Db2 LOGARCHMETH1 or LOGARCHMETH2 parameters must be set to OFF. This setting ensures that IBM Spectrum Protect Plus can set the DISK path for the log backup operation to the vSnap repository.

Refer to the relevant version of the Db2 documentation to review the log backup settings and log backup destinations.

Exceptions

If both LOGARCHMETH parameters are OFF.

When both parameters are OFF, archive logs are not backed up, even if Enable Log Backups is selected. This is because the database is not in recovery mode. You must set one LOGARCHMETH parameter to a value other than OFF, so that the database is in rollforward recoverable mode, and restart the database.

Note: To disable log backups, deselect Enable Log Backups and run the backup job. This setting resets the required LOGARCHMETH parameter to its original value, disabling log backups.

If LOGARCHMETH1 is set to LOGRETAIN, or one is set to USEREXIT.

This configuration supports log backup. IBM Spectrum Protect Plus Db2 application server resets LOGARCHMETH1 to DISK path for the log backup operation to the vSnap repository.

If both LOGARCHMETH parameters are set to DISK, TSM, or VENDOR.

This setting combination causes an error when IBM Spectrum Protect Plus attempts to enable log backups. To fix this, set one of the parameters to OFF, and run the backup job with Enable Log Backups selected.

Note: This setting never backs up logs and prohibits recovery from transactional logs during a restore operation.

Truncating archive log backups

IBM Spectrum Protect Plus automatically deletes older transactional logs after a successful database backup. This action ensures that capacity is not compromised by retention of older log files, which are already saved in the snapshot. The retention of snapshots is defined in the SLA policy chosen. For more information about SLA policies, Defining an SLA Backup Job.

IBM Spectrum Protect Plus does not manage the retention of other archived log locations. Administrators must continue to manage those logs with their existing log retention policies. Db2 provides a command to purge log archives and database backups with a specific timestamp.
