Add a Backup Storage Provider

To enable backup and restore jobs, at least one IBM Spectrum Protect Plus appliance and at least one vSnap server is required. The vSnap server can be located on the IBM Spectrum Protect Plus appliance or on its own appliance, or it can be a physical vSnap installation. Each vSnap server location must be registered so IBM Spectrum Protect Plus recognizes it.

BEFORE YOU BEGIN:

  • Configure a vSnap server. See vSnap Server Administration Reference.

To register Backup Storage:

  1. From the navigation menu, expand System Configuration, then expand Backup Storage. Click Disk.
  2. Click Add . The Storage Properties pane opens.
  3. Populate the fields in the Storage Properties pane:
  4. Hostname/IP
  5. Enter the resolvable IP address or hostname of the backup storage.
  6. Site
  7. Select a site for the backup storage. Available options include Primary, Secondary, or you can create a new site by selecting Add a new site. If more than one Primary, Secondary, or user-defined Backup Storage site is available to IBM Spectrum Protect Plus, the Backup Storage with the largest amount of available storage will be used first.
  8. Username
  9. Enter your username for the backup storage device.
  10. Password
  11. Enter your password for the backup storage device.
  12. Click Save. IBM Spectrum Protect Plus confirms a network connection and adds the backup storage device to the database.
  13. From the Actions menu associated with the newly added backup storage device, select Initialize with Encryption to enable encryption of backup data. To initialize the backup storage device without encryption enabled, select Initialize. The initialization process runs in the background and requires no further user interaction. Note that the process may take 5 to 10 minutes to complete.

To set Storage Options:

  1. From the navigation menu, expand System Configuration, then expand Backup Storage. Click Disk.
  2. Click the Manage icon associated with the vSnap server to add a replication partnership, then expand the Storage Options section. Set the storage options.
  3. Enable Compression
  4. If enabled, each incoming block of data is compressed using a compression algorithm before being written to the storage pool. Compression consumes a moderate amount of additional CPU resources.
  5. Enable Deduplication
  6. If enabled, each incoming block of data (after being compressed if compression is enabled) is hashed and compared against existing blocks in the storage pool. Duplicate blocks are skipped instead of being written to the pool. Deduplication consumes a large amount of memory resources (proportional to the amount of data in the pool) in order to maintain the deduplication table of block hashes, and is disabled by default.
  7. Synchronous Write Mode
  8. Disabling synchronous writes can lead to data loss and silent corruption of backup data if the storage server experiences an abrupt shutdown or reboot during a backup job. Disabling this option is not recommended unless the storage server resides in a stable environment that is adequately secured against hardware/power failures.
  9. Encryption Enabled
  10. This option displays the encryption status of the vSnap server. Encryption can only be enabled during vSnap initialization. This option is for informational purposes only.
  11. Once complete, click Save.

To expand a vSnap storage pool:

To expand a vSnap storage pool, you must first add virtual or physical disks on the vSnap server, either by adding virtual disks to the vSnap virtual machine or adding physical disks to the vSnap physical server. See vSphere documentation for information about creating additional virtual disks. Once complete, perform the following procedure.

Note: Pool expansion must be performed by adding new, unused disks to the pool. Expanding existing disks that are already part of the pool is not supported.

  1. From the navigation menu, expand System Configuration, then expand Backup Storage. Click Disk.
  2. From the Actions menu associated with an existing vSnap server you want to expand, select Rescan.
  3. Click the Manage icon associated with the vSnap server you want to expand, then expand the Add New Disks to Backup Storage section. The Manage vSnap Server section displays.
  4. Add and save the newly added disks. The vSnap pool expands by the size of the added disks.

To establish a replication partnership:

Backup storage replication allows you to perform asynchronous replication of backup data from one vSnap server to another. Establish a partnership through the following procedure.

  1. From the navigation menu, expand System Configuration, then expand Backup Storage. Click Disk.
  2. Click the Manage icon associated with the vSnap server to add a replication partnership, then expand the Partners section.
  3. From the Select Partner field, select a vSnap server with which to establish a replication partnership.
  4. Click Add Partner.

Backup storage replication is enabled through SLA policies by selecting Enable Backup Storage Replication. See Configure SLA Policies.
