Add a Cloud Storage Resource

To enable cloud offload functionality, cloud resources must be registered in IBM Spectrum Protect Plus.

BEFORE YOU BEGIN:

  • Configure cloud resource keys and certificates. See Configure Keys and Certificates.

To register an Amazon S3 cloud resource:

  1. From the navigation menu, expand System Configuration, then expand Backup Storage. Click Cloud.
  2. Click Add Cloud. The Cloud Registration pane opens.
  3. From the Provider field, select Amazon S3, then complete the following fields:
  4. Name
  5. Enter a meaningful name to help identify the cloud resource.
  6. Region
  7. Select the regional endpoint of the cloud resource.
  8. Use existing key
  9. Select a previously entered key. Keys are added through the Keys and Certificates pane. Alternatively, enter key information in the following fields. See Configure Keys and Certificates.
  10. Key name
  11. Enter a meaningful name to help identify the key.
  12. Access key
  13. Enter the AWS access key. Access keys are created through the AWS Management Console.
  14. Secret key
  15. Enter the AWS secret key. Secret keys are created through the AWS Management Console.
  16. Click Get Buckets, then select a bucket associated with the cloud resource. Buckets must be created before registering the cloud resource.
  17. Click Register. The cloud resource is added to the Cloud Servers table.

To register an IBM Cloud Object Storage resource:

  1. From the navigation menu, expand System Configuration, then expand Backup Storage. Click Cloud.
  2. Click Add Cloud. The Cloud Registration pane opens.
  3. From the Provider field, select IBM Cloud Object Storage off premise, then complete the following fields:
  4. Name
  5. Enter a meaningful name to help identify the cloud resource.
  6. Endpoint
  7. Enter the endpoint of the IBM Cloud Object Storage resource.
  8. Use existing key
  9. Select a previously entered key. Keys are added through the Keys and Certificates pane. Alternatively, enter key information in the following fields. See Configure Keys and Certificates.
  10. Key name
  11. Enter a meaningful name to help identify the key.
  12. Access key
  13. Enter the access key.
  14. Secret key
  15. Enter the secret key.
  16. Certificate
  17. Select a method of associating a certificate with the resource.
  18. Upload: Select to browse for the certificate locally, then click Upload.
  19. Copy and paste: Select to enter the name of the certificate, copy and paste the contents of the certificate, then click Create.
  20. Use existing: Select to use a previously uploaded certificate. Certificates are added through the Keys and Certificates pane. See Configure Keys and Certificates.
  21. A certificate is not required when registering a public IBM Cloud Object Storage resource.
  22. Click Get Buckets, then select a bucket associated with the cloud resource. Buckets must be created before registering the cloud resource.
  23. Click Register. The cloud resource is added to the Cloud Servers table.
